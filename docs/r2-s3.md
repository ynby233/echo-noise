# 云存储接入（R2/S3）使用说明

## 字段填写

- 提供方：选择 `R2` 或 `S3`。
- Endpoint：只填基础域名，不带 Bucket 路径。
  - R2 示例：`https://<account-id>.r2.cloudflarestorage.com`
  - S3 示例：`https://s3.<region>.amazonaws.com` 或厂商兼容 S3 的基础域名
- Region：
  - R2 固定填写 `auto`
  - S3 填写对应区域名，如 `us-east-1`、`ap-northeast-1`
- Bucket：你的桶名称，例如 `note`
- Access Key / Secret Key：对象存储的访问密钥对
- 公共访问前缀（可选）：用于直接下载备份的公开地址前缀,如果有如/note 的后缀则会上传到note子文件夹
  - R2 示例：`https://pub-xxxxxxxxxxxxxxxxxxxx.r2.dev`

- Access Key 对应“访问密钥 ID”
- Secret Key 对应“机密访问密钥”

## 自动同步

- 自动同步至云端：开启后，服务端将自动上传备份包 `backup.zip`
- 模式：`即时` 或 `定时`
  - 即时：发布/更新消息后触发，含 15 秒防抖
  - 定时：按“间隔(分钟)”周期执行

## 备份包内容

- SQLite：`database.db` + `data/images/` + `data/video/`
- MySQL/PostgreSQL：图片与视频目录；数据库恢复/下载使用页面的“下载本地备份”进行备份与恢复

## 预签名上传/下载

- 生成上传URL（预签名）：点击“生成”，得到可一次性 PUT 上传 `backup.zip` 的 URL
- 生成下载URL（预签名）：点击“生成”，得到可一次性下载 `backup.zip` 的 URL
- 上传备份到云：将生成的上传 URL 粘贴后点击“上传备份到云”
- 从云恢复备份：将下载 URL 粘贴后点击“从云恢复备份”
- 按配置恢复：当“公共访问前缀”已填写时，系统会尝试从 `<公共前缀>/backup.zip` 恢复

## R2 专项注意

- Endpoint 不要带 Bucket 路径，例如不要写成 `https://...r2.cloudflarestorage.com/note`
- Region 必须为 `auto`
- R2 采用路径风格访问，系统已自动启用

## S3 专项注意

- Endpoint 使用厂商提供的基础域名；是否使用路径风格由系统按 `usePathStyle` 控制
- 若启用了统一访问控制（禁用 ACL），可直接使用系统默认；无需额外 ACL 配置

## 常见问题

- 预签名失败或同步失败：检查 Endpoint 是否为基础域名、Bucket 是否正确、R2 的 Region 是否为 `auto`
- URL 403/签名不匹配：请重新生成预签名 URL，并在有效期内使用；确保密钥正确

## 操作步骤示例（R2）

1. 在 Cloudflare R2 创建 `Access Key` 与 `Secret Key`
2. 建桶 `note`
3. 填写：
   - 提供方：R2
   - Endpoint：`https://<account-id>.r2.cloudflarestorage.com`
   - Region：`auto`
   - Bucket：`note`
   - Access Key / Secret Key：你的密钥
   - 公共访问前缀：`https://pub-xxxxxxxxxxxx.r2.dev`（如需“按配置恢复”）
4. 保存配置，开启“自动同步至云端”与模式
5. 点击“立即同步”进行一次校验
